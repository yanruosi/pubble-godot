【data_src 目录规范 · 2026-07-20】
- 工程源表 ONLY：data_src/xlsx表/*.xlsx、data_src/levels/{关卡id}/*.xlsx
- 禁止在 data_src 下保存/导出/复制 CSV（策划数值草稿用 策划案/…/进度/规则详述.csv、总数值表.csv，誊完写入 xlsx 后转表）
- 禁止把 策划进度 下的 CSV 复制进 data_src（Godot 会为 CSV 生成 .import 污染目录）
- xlsx表/ 与 levels/ 已放置 .gdignore：Godot 编辑器不会导入该目录，避免生成 .translation
- 运行 convert_tables.py / run_convert.bat 会在转表前后自动删除上述目录内除 .xlsx/.xls/.xlsm 外的文件
- 转表输出 ONLY：data/*.json（及 data/levels/{id}/*.json），不要在 data_src 留中间产物
- 若误生成 CSV/.import：删除后只保留 xlsx，并更新 .gitignore 已忽略上述扩展名

所有配表（含 chapters / conditions / levels …）统一下面约定；Excel 另存为 xlsx 时请保留前三行结构。
第 1～3 行固定含义（第 4 行起为数据行）：
第1行：英文字段名（导出 JSON 的 key 建议用这一行）
第2行：类型提示（string / int / float；仅给人看，convert_tables 不解析本行）
第3行：中文说明（列含义与策划约定）
convert_tables.py 会按各表 skip_meta_rows 自动跳过说明行；conditions 表可增加 editor_note 列（策划备注），导出 JSON 时不包含该列。

【chapters 表：删字段或缺列时的连锁反应（游戏侧）】
运行 python convert_tables.py 导出 chapters.json 时，脚本会对每条记录做校验并打印 [警告]。
- id：必填且应为正整数。缺失或无效时无法与存档、levels.chapter_id、选关 setup 对齐。
- order：强烈建议保留。缺失时 card_stack_ui 按章节排序不稳定。
- artist_name / chapter_name：可选；缺失则主页章节卡片标题/副标题为空字符串（chapter_card.gd 使用 .get 默认 ""）。
- icon_id：当前 GDScript 未读取，删了不影响现有玩法；仅留作后续图标资源绑定。
- condition_id：可选；若 JSON 中缺少该键（常见原因：CSV 表头只有 5 列、列未对齐）：level_select_page 章节解锁按钮文案退化为「解锁章节」，且不扣应援棒；chapter_list_ui.smart_scroll_to_first_locked 将该章视为 condition_id=0（is_condition_met 恒通过）。若键存在且值为 0，表示策划主动配置「章节无应援棒门槛」，与「缺列」不同。
改表后务必重新导出 chapters.json，并确认控制台无 [警告]。

【levels 全局表】
源文件：data_src/levels.csv -> data/levels.json
- level_id：全局唯一，建议与单关目录名一致（如 ch1_l01）。
- data_dir：单关配表子目录名，对应 data_src/levels/{data_dir}/ 与 data/levels/{data_dir}/。
- scene_path：Godot 场景 res:// 路径；序章 ch1_l01 指向 idol 玩法场景。
- unlock_condition_id：int，对接 conditions.json（0 表示无额外条件）。
- 已废弃：寻物时代的 props.csv / level_props.csv 不再转表。

【单关配表目录 data_src/levels/{level_id}/】
每关四表（xlsx 中文文件名），转表后输出到 data/levels/{level_id}/：
- 关卡配置表.xlsx -> level.json（通常 1 条：真相、槽位开关、required_slot、场景底图与坐标基准）
- 词条表.xlsx -> vocab.json（可拖入词条库）
- 热点表.xlsx -> hotspots.json（parent_id 树；type: normal/collect/modal/panel）
- 槽位表.xlsx -> slots.json（slot_type: scroll / identity / mapping；卷轴用 prefix_text/suffix_text）

热点坐标：按 level.art_base_width × art_base_height 设计稿像素（默认 1920×1080），运行时缩放至视口。

【feed_posts 表（动态页帖子）】
源文件：data_src/feed_posts.xlsx（或同名 csv），导出 data/feed_posts.json；已在 convert_tables.py TABLES 注册。
- post_id：必填，全局唯一。
- chapter_id：对齐 chapters.id；用于点头像进关卡选择页等跳转。
- level_id：对齐 levels.level_id；发帖时间在运行时由 levels.text2 读取，不必在本表重复。
- text：帖子正文。
- image_path / avatar_path：res:// 资源路径；占位可用 art/feed 下占位图。
- 动态页「推荐/关注」是否展示由系统解锁状态动态判断，不再依赖 tab 字段。
- 艺人名由章节表（chapters.artist_name）读取，不再依赖 feed_posts.artist_name 字段。
- 推荐/关注排序由程序规则决定（recent/open 状态 + 章节/关卡 order），不再依赖 sort_weight 字段。
